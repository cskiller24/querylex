    1. `select_simple` — Basic SELECT: MySQL/PostgreSQL/MariaDB/SQLite: `SELECT {columns} FROM {table} LIMIT 3`, MSSQL: `SELECT TOP 3 {columns} FROM {table}`. All engines: want `""` (success).

    3. `select_with_aggregate` — GROUP BY + COUNT: MySQL/MariaDB: `SELECT dept_no, COUNT(*) as cnt FROM dept_emp GROUP BY dept_no`, PostgreSQL: `SELECT dept_no, COUNT(*) as cnt FROM dept_emp GROUP BY dept_no` (different dataset — Pagila uses `film_category`), MSSQL: `SELECT CustomerID, COUNT(*) as cnt FROM Orders GROUP BY CustomerID`, SQLite: `SELECT GenreId, COUNT(*) as cnt FROM Track GROUP BY GenreId`. Want `""`.

    4. `select_with_subquery` — Subquery in WHERE: Engine-specific subquery SQL using the appropriate dataset tables. Want `""`.

    5. `select_with_cte` — CTE (WITH clause): MySQL 8.0+ and MariaDB 10.2+ support CTEs. PostgreSQL, MSSQL, SQLite support CTEs. Provide engine-appropriate CTE SQL. Want `""`. For engines that don't support CTEs: `want` → `"INVALID_SQL"` (e.g., older MySQL but 8.4 supports CTEs so all engines should pass).

    6. `select_order_by` — ORDER BY: Standard SQL works across all engines. Want `""`.

    7. `select_limit_variation` — LIMIT vs TOP n: MySQL/PostgreSQL/MariaDB/SQLite: `LIMIT 5 OFFSET 2`, MSSQL: `SELECT ... ORDER BY ... OFFSET 0 ROWS FETCH NEXT 3 ROWS ONLY` or `SELECT TOP 3 ...`. Want `""`.

    8. `type_coercion_string` — String concatenation: MySQL/MariaDB: `CONCAT(first_name, ' ', last_name)`, PostgreSQL: `first_name || ' ' || last_name`, MSSQL: `first_name + ' ' + last_name`, SQLite: `first_name || ' ' || last_name`. Want `""`.

    9. `type_coercion_numeric` — Numeric expression: Standard arithmetic works across engines. MySQL/MariaDB: `SELECT emp_no + 1000 FROM employees LIMIT 3`, etc. Want `""`.

    10. `invalid_syntax` — Deliberately broken SQL: All engines: `"SYNTAX ERROR GARBAGE XYZ"`. Want `"INVALID_SQL"`.

    11. `table_not_found` — Non-existent table: All engines: `"SELECT * FROM nonexistent_table_xyz123"`. Want `"TABLE_NOT_FOUND"`.

    12. `column_not_found` — Non-existent column on valid table: Engine-specific valid table name with nonexistent column. MySQL/MariaDB: `"SELECT nonexistent_col_xyz123 FROM employees"`, PostgreSQL: same on `actor`, MSSQL: same on `Customers`, SQLite: same on `Album`. Want `"COLUMN_NOT_FOUND"`.
