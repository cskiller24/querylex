## Requirements at Ship

All 46 v1 requirements were addressed across 3 phases. The traceability table reflects final status.

### Build Unification

- [x] **BLD-01**: Remove standalone `querylex-add-db` and `querylex-stats` binary builds from goreleaser config
- [x] **BLD-02**: Remove standalone binary targets from Makefile
- [x] **BLD-03**: Delete `cmd/querylex-add-db/` and `cmd/querylex-stats/` directories
- [x] **BLD-04**: Clean up CI workflow to only build single `querylex` binary
- [x] **BLD-05**: Verify all existing functionality still works via `querylex add-db` and `querylex workspace-stats` subcommands

### Docker Compose Infrastructure

- [x] **DKR-01**: Single `compose.yaml` with profiles for each database (mysql, postgresql, mariadb, mssql)
- [x] **DKR-02**: Healthchecks with `condition: service_healthy` for all database containers
- [x] **DKR-03**: tmpfs storage for ephemeral test data
- [x] **DKR-04**: Random host ports to avoid conflicts with local instances
- [x] **DKR-05**: Memory limits per container (especially MSSQL at 2GB)
- [x] **DKR-06**: Version-pinned Docker images (no `:latest`)
- [x] **DKR-07**: Makefile targets: `compose-up-{db}`, `compose-down`, `build-test`
- [x] **DKR-08**: SQLite testing without Docker (file-based, pure Go driver)

### Test Helpers

- [x] **HLP-01**: `test/testhelper` package with `Connect*(t)` functions per database
- [x] **HLP-02**: Environment-variable-driven DSN configuration with sensible defaults
- [x] **HLP-03**: Port wait with retry and backoff
- [x] **HLP-04**: `t.Cleanup()` registration for connection teardown
- [x] **HLP-05**: FixtureRunner for loading targeted per-test SQL fixtures
- [x] **HLP-06**: CLI execution helper (`os/exec` wrapper with timeout, stdout/stderr, exit code)
