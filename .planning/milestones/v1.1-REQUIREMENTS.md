# Requirements: QueryLex

**Defined:** 2026-06-07
**Core Value:** AI agents can reliably introspect any supported database, generate correct SQL from natural language descriptions, and optimize queries — all through a single CLI tool with structured machine-readable output.

## v1 Requirements

Requirements for v1.1 Cleanup milestone. Each maps to roadmap phases.

### AI Package Removal

- [x] **AIPKG-01**: Delete `internal/ai/` package (11 source + test files: client, config, chat, embed, prompt, tokens + their `_test.go` counterparts)
- [x] **AIPKG-02**: Delete AI CLI handler files (`internal/cli/run_sql.go`, `internal/cli/run_sql_test.go`, `internal/cli/run_optimize.go`, `internal/cli/run_ai_config.go`, `internal/cli/run_ai_config_test.go`)
- [x] **AIPKG-03**: Remove AI command registrations from `internal/rootcmd/root.go` (`sqlCmd`, `optimizeCmd`, `aiConfigCmd` definitions, `AddCommand` calls, and their function imports)

### AI Surgical Edits

- [x] **AIEDIT-01**: Remove `AIConfig` field from `PreflightResult` struct and `ai.ResolveAIConfig` call from `internal/cli/preflight.go`
- [x] **AIEDIT-02**: Remove 4 AI error code constants (`ErrCodeAIServiceUnavailable`, `ErrCodeAIConfigMissing`, `ErrCodeAIGenerationFailed`, `ErrCodeAIContextOverflow`) and their descriptions from `internal/format/error.go`
- [x] **AIEDIT-03**: Remove `envVarAIKey` constant and `case "ai-key"` branch from `Retrieve()` and `Available()` in `internal/credentials/env.go`
- [x] **AIEDIT-04**: Remove `kind == "ai"` branch from `promptEncryptedFilePassphrase` in `internal/cli/passphrase.go`
- [x] **AIEDIT-05**: Remove AI key from `setEnvForTest` test helper and simplify test setup in `internal/credentials/factory_test.go`

### E2E Infrastructure Removal

- [x] **E2ERM-01**: Delete entire `test/` directory hierarchy (~50+ Go test files, testdata, scripts)
- [x] **E2ERM-02**: Delete `compose.yaml` and `Dockerfile.mssql`
- [x] **E2ERM-03**: Delete `.github/workflows/e2e.yml` CI workflow file
- [x] **E2ERM-04**: Remove E2E/Docker/compose-related targets from `Makefile` (ci-setup, build-test, compose-up-*, compose-down, test-e2e-*, test-e2e-all, release)

### Memory Dead Code Cleanup

- [x] **MEMCLN-01**: Remove dead AI-related code from `internal/memory/` (`EmbeddingMetadata` struct, `EmbeddingVectors` map field, `SearchWithEmbeddings` function, `cosineSimilarity` function, `embeddingsActive` parameter branch in `ComputeSimilarity`, `EMBEDDINGS_UNAVAILABLE` warning)

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Re-adding AI integration | Intentional removal, reverse requires explicit decision |
| Adding new database engines | Not requested |
| GUI or web interface | CLI-only by design |
| Schema indexing pipeline changes | Retained as-is — used by non-AI commands |
| Heuristic analysis changes | Retained as-is — used by explain command |
| Credential store restructuring | Only AI key branch removed, DB credential storage untouched |
| `selectCredentialStore` dead code in `run_adddb.go` | Pre-existing, not part of AI/E2E removal scope |
